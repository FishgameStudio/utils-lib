# Documentation for SHA‑256 Hash Module
Function usage: Implements the standard **SHA‑256 cryptographic hash algorithm**, converts any input string into a fixed 64‑character hexadecimal hash string.


# Documentation of constant `K`
Function usage: SHA‑256 standard **64 round constants** (cube roots of first 64 primes).
Definition:
```cpp
const uint32_t K[64] = { ... };
```
**Why needed**: Fixed by SHA‑256 standard to ensure hash uniqueness and security.


# Documentation of function `rotr`
Function usage: **Cyclic right shift** (core bit operation in SHA‑256).
Implementation:
```cpp
inline uint32_t rotr(uint32_t x, int n) {
    return (x >> n) | (x << (32 - n));
}
```
**Logic**: Shift right `n` bits, overflow bits move to left side.
**Why needed**: SHA‑256 requires bit mixing via cyclic rotation.


# Documentation of function `sig0`
Function usage: SHA‑256 lower sigma 0 (small message schedule function).
Implementation:
```cpp
inline uint32_t sig0(uint32_t x) {
    return rotr(x, 7) ^ rotr(x, 18) ^ (x >> 3);
}
```
**Logic**: Rotate + XOR + shift → mix bits for message expansion.


# Documentation of function `sig1`
Function usage: SHA‑256 lower sigma 1 (small message schedule function).
Implementation:
```cpp
inline uint32_t sig1(uint32_t x) {
    return rotr(x, 17) ^ rotr(x, 19) ^ (x >> 10);
}
```
**Logic**: Same purpose as `sig0`, different rotation parameters.


# Documentation of function `ep0`
Function usage: SHA‑256 upper sigma 0 (big compression function).
Implementation:
```cpp
inline uint32_t ep0(uint32_t x) {
    return rotr(x, 2) ^ rotr(x, 13) ^ rotr(x, 22);
}
```
**Logic**: Strong bit mixing for compression loop.


# Documentation of function `ep1`
Function usage: SHA‑256 upper sigma 1 (big compression function).
Implementation:
```cpp
inline uint32_t ep1(uint32_t x) {
    return rotr(x, 6) ^ rotr(x, 11) ^ rotr(x, 25);
}
```
**Logic**: Same purpose as `ep0`, different rotation parameters.


# Documentation of function `ch`
Function usage: **Choice** function in SHA‑256 compression.
Implementation:
```cpp
inline uint32_t ch(uint32_t x, uint32_t y, uint32_t z) {
    return (x & y) ^ (~x & z);
}
```
**Logic**: Choose output from `y` or `z` based on bits in `x`.


# Documentation of function `maj`
Function usage: **Majority** function in SHA‑256 compression.
Implementation:
```cpp
inline uint32_t maj(uint32_t x, uint32_t y, uint32_t z) {
    return (x & y) ^ (x & z) ^ (y & z);
}
```
**Logic**: Output the majority bit among three inputs.


# Documentation of function `sha256`
Function usage: Public SHA‑256 interface. Input string → output 64‑char hex hash.
Implementation:
```cpp
std::string sha256(const std::string& input)
```
**Core steps**:
1. **Initialize 8 standard hash values** (from SHA‑256 standard).
2. **Preprocess input**:
   - Append `0x80` padding.
   - Fill `0x00` to reach 448 mod 512 bits.
   - Append original message length (64‑bit big‑endian).
3. **Process in 512‑bit (64‑byte) blocks**:
   - Split block into 16 × 32‑bit words.
   - **Expand** to 64 words using `sig0` / `sig1`.
4. **64 rounds of compression**:
   - Use `ep0`, `ep1`, `ch`, `maj`, `K` constants.
   - Update working variables `a,b,c,d,e,f,g,hh`.
5. **Update final hash** values.
6. **Convert hash to hex string** (64 characters, lowercase, zero‑padded).

**Return**: 64‑character SHA‑256 hex string.
**Example**:
`"hello"` → `"2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824"`
